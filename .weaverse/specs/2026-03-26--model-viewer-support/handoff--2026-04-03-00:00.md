### Key Analysis

**Why MODEL_3D has 0 height (original bug):**
- `<model-viewer>` has no intrinsic dimensions (unlike `<img>`)
- When alone in a grid row (via `col-span-2`), the CSS height chain is circular: cell height depends on content, content `height: 100%` depends on cell = collapses to 0
- In `media-item.tsx` line 85: when `imageAspectRatio === "adapt"`, the model gets `{ height: "100%" }` which can't resolve without a parent height

**Why behavior differs between direct load and variant switch:**
- `variant-media-group.ts` groups media by extracting option values from **image filenames** (lines 28-72)
- `MODEL_3D` media and images with non-matching filenames go to `ungrouped` (line 119)
- `ungrouped` items are appended to EVERY variant's `displayMedia` (line 142: `[...matched, ...ungrouped]`)
- Direct load with brown (2 media): only brown's media in the loader data = 2 items, model shares a row with the image
- Switch from black to brown: `allMedia` includes ALL variants' media. Unmatched images from other variants leak into `ungrouped`, inflating brown's `displayMedia` count. Model ends up alone in its row with `col-span-2`

## Suggested Fix Approach
Two-pronged fix needed:
1. **`variant-media-group.ts`**: Don't blindly append all `ungrouped` to every variant. Either: assign ungrouped media only to the variant they belong to (via variant's `image` reference), or exclude non-image media from grouping entirely and always include them
2. **`media-grid.tsx`**: Only add `aspect-video` (or `min-h-*`) to MODEL_3D cells when they are actually alone in their grid row (check `col-span-2` condition), not unconditionally
